1.Регистрация пользователя 
  - Отправляем - массив с данными POST - interface UserData
  - получаем - массив с присвоенным id
2. Проверка залогиненного пользователя
  - отправляем массив на проверку interface UserDataRequest
  - ответ сервера - true - доступ к функционалу сайта interface UserDataResponse
                  - false - сообщение об ошибке
3. Восстановление пароля 
  - отправляем на проверку email 
  - ответ сервера - сгенерированная ссылка на данную почту- переход - вводит новый пароль
4. Внесение информации о пользователе 
   - отправляем массив с данными, введенными пользователем - interface UserProfileModel
   - получаем массив - отрисовываем на странице
5. Отображение профиля
   - get запрос - получаем массив 
6. Изменение профиля 
   - отправляем изменненную инфо  interface UserChangeDataRequest
   - получаем обновленную инфо  interface  UserChangeDataResponse 
6. Подписаться на обновление пользователя 
   - отправляем id пользователя в 2 массива- на кого я подписан - и в массив подписчиков пользователей 
   - получаем массив постов данного юзера - interface Post
7.  Отображение кол-во подписчиков 
   - отправляем id пользователя -  interface SubscribersRequest
   - получаем length массива подписчиков - SubscribersResponse
8. Отображение кол-во подписок
   - отправляем id пользователя  -  interface  SubscribtionsRequest 
   - получаем length массива, на кого пользователь подписан interface SubscribtionsResponse
9.  Отображение списка подписчиков 
   - отправляем id пользователя - interface SubscribersRequest
   - получаем массив подписчиков - SubscribersResponse
10. Отображение списка подписок
   - отправляем id пользователя - interface  SubscribtionsRequest 
   - получаем массив, на кого пользователь подписан - interface SubscribtionsResponse
11. Добавление поста 
   - пушим в массив posts - interface PostAddRequest, 
   - получаем обновленный массив interface PostAddResponse 
12. Редактирование поста
   - отправляем отредактированную инфо - interface Post
   - получаем обновленный массив (по id находим конкретный пост)
13. Удаление поста 
   - отправляем id posta interface Post
   - получаем обновленный массив 
14. Добавление комментария 
    - отправляем комментарий - interface CommentAddRequest
    - получаем - обновленный пост - interface  CommentAddResponse
15. Отображения постов пользователей, на кого подписаны
   - отправляем - массив - interface SubscribtionsResponse
   - получаем - массив всех постов, на кого подписаны (фильтрация по дате)
16. Поиск новых юзеров 
    - получаем массив пользователей  (сразу 20 рандомных), вводим name, surname
    - получаем отфильтрованный массив пользователей
